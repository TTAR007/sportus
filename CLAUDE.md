## Project Overview

**Sportus** is a Flutter mobile app for iOS and Android that lets users create, update, delete, and join sport activities. The backend is Firebase (Firestore + Firebase Auth).

## Architecture Rules

These are **non-negotiable**. Never break them.

### State Management — Riverpod only
- Use `flutter_riverpod` exclusively. Never use `setState`, `ChangeNotifier`, `BLoC`, or the `provider` package.
- All providers live in `lib/providers/` as top-level `final` variables.
- Use `AsyncNotifier` for async data (list, detail). Use `Notifier` for form/UI state.
- Screens read state with `ref.watch(...)`. Side-effects use `ref.read(...).someMethod()`.
- Providers must never receive or store a `BuildContext`.

### Navigation — go_router only
- Use `go_router` exclusively. Never call `Navigator.push()`, `Navigator.pop()`, or `Navigator.of(context)`.
- All routes are defined in `lib/router/app_router.dart`. No inline `GoRoute` definitions elsewhere.
- Navigate with `context.go('/path')` or `context.push('/path')`.
- Pass complex objects via Riverpod state, not route `extra`, except for simple string IDs.

### Data Layer — repository only
- All Firestore reads/writes go through `ActivityRepository`. No widget or provider imports `cloud_firestore` directly.
- Repository methods return `Either<AppException, T>` using a sealed class.
- Soft-delete only — set `deletedAt` to `FieldValue.serverTimestamp()`. Never call `.delete()` from the client.
- Always use `FieldValue.serverTimestamp()` for `createdAt` and `updatedAt`.

### Models — Freezed immutable
- All data models use `@freezed`. Never mutate a field directly — use `.copyWith(...)`.
- Run `dart run build_runner build --delete-conflicting-outputs` after any model change.

### UI Layer
- All colors come from `Theme.of(context).colorScheme` or `Theme.of(context).extension<AppColorScheme>()!`. Zero hardcoded hex values.
- All text styles come from `AppTextStyles`. Zero inline `fontSize` or `fontWeight`.
- Reusable widgets for one feature → `lib/features/<feature>/widgets/`. Cross-feature → `lib/shared/widgets/`.
- Loading states use `ShimmerCard` skeleton. Never show a bare `CircularProgressIndicator` as the only full-screen feedback.
- Destructive actions always show a confirmation `AlertDialog` before executing.
- All files stay under 300 lines. Split if larger.

### Logging
- Never use `print()`. Use `debugPrint()` inside `assert(...)` for debug-only output.

```dart
String hostId                // Firebase Auth UID

List<String> participantIds  // UIDs of joined users

DateTime updatedAt

```

| Token | Size | Weight |

### Spacing
4px grid only. Use `Gap` widget. Screen horizontal padding: `16px`.

### Components
- **Cards**: `BorderRadius.circular(16)`, `elevation: 0`, border `cs.outline`, background `cs.surface`
- **Screen bg**: `cs.surfaceContainerLowest`

- **Touch targets**: minimum `44×44px`

## Testing

- **Unit tests** — models, validators, repository logic → `test/unit/`
- **Provider tests** — Riverpod notifiers → `test/providers/`
- **Widget tests** — must cover both light and dark mode → `test/widgets/`
- **Integration tests** — full user flows → `integration_test/`

Widget test theme helper (use this in every widget test file):
```dart
Widget pumpWithTheme(Widget child, {bool dark = false}) {
  return ProviderScope(
    child: MaterialApp(
      theme: AppTheme.light(),
      darkTheme: AppTheme.dark(),
      themeMode: dark ? ThemeMode.dark : ThemeMode.light,
      home: Scaffold(body: child),
    ),
  );
}
```

Provider test pattern — Riverpod 3 has testing built in, no extra package needed:
```dart
// ✅ Riverpod 3 — use ProviderContainer.test()
final container = ProviderContainer.test(overrides: [
  activityRepositoryProvider.overrideWithValue(mockRepo),
]);
// Container is automatically disposed after the test — no addTearDown needed

// ❌ Wrong — riverpod_test package does not exist for Riverpod 3
```

## Common Pitfalls

| Pitfall | Correct approach |
|---------|-----------------|
| Using `riverpod_test` package | It doesn't exist for Riverpod 3 — use `ProviderContainer.test()` built in |
| Hardcoded `Color(0xFF...)` in a widget | Use `cs.*` or `ext.*` from theme |
| Assuming white/black — works in light, breaks in dark | Always use `cs.surface` / `cs.onSurface` |
| `import 'package:cloud_firestore/...'` in a widget/provider | All Firestore calls go through `ActivityRepository` |
| `Navigator.push(...)` | Use `context.go()` or `context.push()` |
| Mutating a Freezed model | Use `.copyWith(...)` |
| Forgetting `build_runner` after model change | `dart run build_runner build --delete-conflicting-outputs` |
| Not handling all `AsyncValue` states | Always use `.when(data:, loading:, error:)` |
| Hard-deleting a Firestore document | Set `deletedAt` — soft delete only |
| Joining without capacity check | Check `currentParticipants < maxParticipants` in repository before writing |
| Inline `fontSize` / `fontWeight` in a widget | Use `AppTextStyles` tokens |
